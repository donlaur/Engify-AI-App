## CONTENT REQUIREMENTS

### Tone & Style

- **Professional but approachable** - Not overly technical, but not dumbed down
- **Engineer-focused** - Write for developers, not marketers
- **Practical** - Real examples, not theory
- **Comparative** - Always compare with alternatives
- **Actionable** - "Use X for Y" recommendations

### Content Structure

**For Model Pages** (`/learn/ai-models/[slug]`):

1. **Hero Section**: Model name, provider, status badge (Active/Deprecated/Sunset)
2. **Overview**: What is this model? When was it released?
3. **Specifications**: Context window, pricing, capabilities table
4. **EOL Status**: Deprecation date, sunset date, replacement model (if applicable)
5. **Use Cases**: When to use this model, when NOT to use it
6. **Comparison**: How it compares to similar models
7. **Pricing Deep Dive**: Cost analysis with examples
8. **Related**: Links to alternatives, comparisons, related prompts

**For Tool Pages** (`/learn/ai-tools/[slug]`):

1. **Hero Section**: Tool name, category, rating, pricing badge
2. **Overview**: What is this tool? Who makes it?
3. **Features**: Comprehensive feature list
4. **Pricing**: Free tier, paid tiers, enterprise options
5. **Pros & Cons**: Balanced, honest evaluation
6. **Comparison**: How it compares to alternatives
7. **Use Cases**: Best for X, not ideal for Y
8. **Reviews**: Aggregate rating, common feedback
9. **CTA**: Try the tool (affiliate link)
